# Лабораторна робота №3: Інфраструктура як код (IaC) з Terraform та AWS

Цей проєкт демонструє використання Terraform для автоматизованого розгортання базової мережевої інфраструктури (VPC, Subnets, Internet Gateway, Route Tables) та вебсервера (EC2 інстанс з Apache) у хмарі Amazon Web Services.

## Передумови (Prerequisites)

Перед початком роботи з цим проєктом переконайтеся, що у вас налаштовано:
- **AWS CLI (v2)**: Встановлено та автентифіковано (`aws configure`) з доступом до регіону `eu-central-1`.
- **Terraform**: Встановлено версію `1.10.0` або новішу.
- **Amazon S3**: Створено захищений бакет для віддаленого зберігання файлу стану (Remote Backend) з увімкненим версіонуванням та шифруванням.
- **Git**: Для контролю версій та синхронізації з репозиторієм.

## Інструкції з використання

Для управління інфраструктурою використовуйте наступні команди Terraform:

### 1. Ініціалізація
Завантажує необхідні плагіни провайдера (AWS) та підключається до S3 бакету для віддаленого стану.

```bash
terraform init
```

### 2. Планування
Генерує план виконання, показуючи, які саме ресурси будуть створені або змінені в AWS, без їх фактичного застосування.

```bash
terraform plan
```

### 3. Розгортання (Застосування)
Виконує код та розгортає описану інфраструктуру в хмарі. Система попросить підтвердити дію (введіть `yes`).

```bash
terraform apply
```

### 4. Знищення (Декомісіювання)
Видаляє всі ресурси, створені цим конфігураційним файлом, щоб уникнути зайвих витрат в AWS

```bash
terraform destroy
```
